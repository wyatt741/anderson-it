# anderson-it — Project Entry

Public marketing site for **Anderson Technologies** (the IT / managed-services + on-demand
tech-support brand). Served by GitHub Pages and **LIVE at https://andersontechsupport.com**.
Static HTML/CSS/vanilla JS, generated by a single Python script. No build framework.

This is a **separate business and site** from the commissioning firm in `../andersontech-site`
(data-center commissioning, currently parked at wyatt741.github.io/andersontech-site,
destined for andersoncx.com). Do not conflate the two.

## Files
| Path | Purpose |
|------|---------|
| `build.py` | Emits EVERY page (index, business, support, ai, about, reviews, contact, thanks, 404) + sitemap.xml/robots.txt. Edit content HERE, then run `py build.py`. Never hand-edit generated HTML. |
| `styles.css` / `app.js` | Single stylesheet + script, cache-busted via `?v=N` (`CSSV` in build.py; the `app.js?v=N` in `footer()`). Bump on any change. |
| `assets/` | Real licensed photos (Pexels) + logo/favicons; licensing recorded in `LICENSES.md`. |
| `CNAME` | andersontechsupport.com |
| `docs/` | Git-ignored working docs (project + session state) — local only, NOT published (public repo). |

## Conventions
- No em dashes anywhere. No fabricated stats / pricing / testimonials / certs. Reviews render only owner-supplied real ones.
- Dark theme is the default (Ethereal Glass); light via toggle, persists choice (FOUC script defaults to dark, no longer OS-based).
- Bump `?v=N` for BOTH `styles.css` (`CSSV`) and `app.js` on any CSS/JS change (caching bit us once).
- Deploy = `git push origin master`. Repo is **PUBLIC** (Pages) — never commit internal notes or personal emails.
- Contact form uses FormSubmit.co -> info@andersontechsupport.com. The endpoint **MUST stay lowercase**
  (changing case forces re-activation). Photo upload + stay-on-page use a hidden `<iframe>` target.
  See `docs/` for the FormSubmit + Microsoft 365 gotchas.
